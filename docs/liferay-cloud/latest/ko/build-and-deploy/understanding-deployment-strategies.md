# 배포 전략 이해

서비스의 배포 전략에 따라 새 버전이 초기화되는 방법이 결정됩니다. Liferay Cloud는 서비스에 대해 두 가지 배포 전략을 제공합니다. `RollingUpdate` 및 `Recreate`.

## 배포 전략의 차이점

사용자는 애플리케이션을 항상 사용할 수 있기를 기대하고 개발자는 애플리케이션의 새 버전을 하루에 여러 번 배포해야 합니다. Liferay Cloud에서 이것은 `RollingUpdate` 전략으로 수행됩니다. 롤링 업데이트를 사용하면 인스턴스를 새 인스턴스로 점진적으로 업데이트하여 다운타임 없이 배포를 수행할 수 있습니다.

| 배포 전략           | 장점                              | 단점                                          |
|:--------------- |:------------------------------- |:------------------------------------------- |
| `RollingUpdate` | 새 버전은 점진적으로 출시되어 가동 시간을 극대화합니다. | 전체 롤아웃은 인스턴스 수에 따라 추가 시간이 걸릴 수 있습니다.        |
| `Recreate`      | 애플리케이션 상태를 한 번에 완전히 갱신          | 발생하는 가동 중지 시간은 애플리케이션의 종료 및 부팅 시간에 따라 다릅니다. |

```{important}
'Recreate' 전략으로 정의된 배포는 최신 버전으로 다시 만들기 전에 실행 중인 모든 인스턴스를 종료합니다.
```

일반적으로 `RollingUpdate` 전략이 권장되는 접근 방식입니다. `Recreate` 전략은 비프로덕션 환경이나 매우 특정한 경우에 사용할 수 있습니다.

## 배포 전략 구성

기본적으로 Liferay Cloud의 서비스는 `RollingUpdate` 전략을 사용하도록 미리 구성되어 있습니다. 각 서비스의 배포 전략은 서비스의 [`LCP.json`](../reference/configuration-via-lcp-json.md)에서 `전략` 변수를 통해 구성할 수 있습니다.

```json
{
    "id": "myservice",
    "strategy": {
        "type": "Recreate"
    }
}
```
