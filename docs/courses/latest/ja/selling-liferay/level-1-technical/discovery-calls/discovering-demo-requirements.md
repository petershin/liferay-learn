# ディスカバリー・コール - デモ・ディスカバリー

**概要**

* ディスカバリー・コールは以下の目的で使用される。
  * クオリファイと基本的なディスカバリーの実行
  * 次のデモで何を見せる必要があるのかを発見する。
* セールス・エンジニアは両方のフェーズに関わるべきである。
* デモ・ディスカバリーの焦点は、それを理解することである。
  * オーディエンス
  * ペルソナ
  * 必要な条件
  * UX & ユーザージャーニーマッピング
  * 技術スタック
  * 現在のシステム
  * サイズ

## デモの要件を発見する

デモの要件を決定するのはチームワークである。 オポチュニティの特定、オポチュニティの資格認定、その他オポチュニティに関わるすべての人が、その一翼を担うべきである。 BDR、SDR、アカウント・マネージャー、アカウント・エグゼクティブの全員が、デモで何が必要かをセールス・エンジニアに伝えます。 したがって、一人で、あるいはサイロで仕事をするのではなく、最初からチームとして仕事をすることが重要なのだ。

次に、適切な質問を学ぶことで、見込み客が要件について詳細を共有し、その要件に最適なデモを構築または調整するのに役立つ。

主に以下のことを確立する必要がある：

* 聴衆-誰が通話に参加するのか、その役割や肩書きは？
* ペルソナ
* 必要な条件
* UX, ユーザージャーニーマッピング
* 技術スタック
* 現在のシステム
* サイズ

### オーディエンス

デモの準備で最も重要なのは、誰がデモに参加するかを決めることだろう。

誰が参加するかが決まったら、次のステップは、彼らの役割と役職、プロジェクト内での責任を理解することだ。

### ペルソナ

オーディエンスを知ることは、次のステップであるペルソナデモの準備に役立ちます。 デモに聴衆に沿ったペルソナが含まれていれば、大きなインパクトが生まれ、デモがより親しみやすいものになる。

また、ビジネス要件も参照する必要があります。たとえば、要件がイントラネットのユースケースの場合、デモには顧客やエージェントなどのペルソナを含めるべきではありません。

### 必要な条件

ビジネスの成果に重点を置き、実装の詳細にはあまり目を向けない。 機会が進むにつれて、最終的にどのような実装になるかを議論する機会が多くなるでしょうが、この段階ではLiferayが正しい選択であるという自信を築くことが重要なので、Liferayがすぐに使える機能、あるいはカスタマイズによってどのように要件に対応するかに焦点を当てます。

### UX, ユーザージャーニーマッピング

オーディエンス、ペルソナ、要件を理解することで、魅力的なユーザー体験とともに、デモに必要なユーザー・ジャーニーを作成することができます。 この学習パスの後半のセッションは、魅力的で親近感のあるユーザー体験を構築するための支援に重点を置いている。

Liferayは広大なプラットフォームであり、ユーザージャーニー、ユーザエクスペリエンス、デモのためのデモスクリプトが定義されていないと、プラットフォームの特定の機能について話している間に混乱し、迷子になりやすくなります。 これらを持つことで、デモは要件に沿ったものとなり、また与えられた時間を最大限に活用することができる。

### 技術スタック

なぜなら、オポチュニティがすでに技術的にも機能的にも適格である場合、デモを構築する際にこの理解を活用することで、余裕を持たせることができるからである。

もし希望がなければ、デモはLiferayがスタックであると仮定し、Liferayが他のシステムにどのように統合できるかを中心に話を進めることができます。 もし希望があれば、それに応えなければならない。それが答えられなければ、契約違反になりかねないからだ。 例えば、見込み客は、既存のスタックにLiferayのような新しい技術を導入することが実際にどのように機能するのかを心配しているかもしれません。セールスエンジニアがこれに対処しなければ、Liferayが適切なソリューションになり得るという確信が低くなってしまうかもしれません。

多くのことはソリューションアーキテクチャに依存しますが、最初の数回のミーティングでテクノロジースタックについて話し合うことは、Liferayと見込み客の双方が期待値を正しく設定するのに役立ちます。 Liferayは多くのテクノロジーに不可知論的であり、バックエンドとフロントエンドのテクノロジーを自由に選択することもできる。

### 現在のシステム

テクノロジースタックと同様に、Liferayが既存のシステムにどれだけシームレスに統合できるかが、一連の質問となる。 そして、統合する必要のある既存の技術が何であるかを知り、Liferayがどのように適合できるかを知ることが重要です。

### サイズ

SaaSであれ、PaaSであれ、オンプレミスであれ、インプリメンテーションのサイジングを正しく行うことは、オポチュニティの初期段階において非常に重要である。
