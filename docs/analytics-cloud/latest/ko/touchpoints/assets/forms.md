# 양식

[양식 소개](https://learn.liferay.com/w/dxp/process-automation/forms/introduction-to-forms) 은 기업을 위한 중요한 직접 데이터 수집 도구입니다. 귀하의 웹 양식이 귀중한 정보를 제공하고 있습니까, 아니면 사용자를 외면하고 있습니까? Analytics Cloud는 중요한 통찰력을 제공합니다.

![Forms metrics are presented in a table with submission, view, abandonment, and completion time statistics.](forms/images/01.png)

사이트 양식 목록이 다양한 측정항목과 함께 표시됩니다. 이 데이터를 CSV 파일로 다운로드하려면 **보고서 다운로드** 를 클릭하세요. 자세한 내용은 [보고서 다운로드](../../reference/downloading-reports.md) 참조하세요.

## 방문자 행동

모든 자산 유형과 마찬가지로 방문자 행동 차트는 페이지 상단에 있으며 여러 추세선이 있는 선 그래프를 제공합니다. 이 대시보드 보기를 PDF 파일로 다운로드하려면 **보고서 다운로드** 를 클릭하세요. 자세한 내용은 [보고서 다운로드](../../reference/downloading-reports.md) 참조하세요.

방문자 행동이라는 첫 번째 차트에는 네 가지 중요한 측정항목이 포함되어 있습니다.

1. 총 제출물
1. 총 조회
1. 이탈률
1. 평균 완료 시간

![Behavior is charted using a line graph to identify trends across a variety of metrics.](forms/images/02.png)

### 제출

제출은 양식에서 제출 버튼을 클릭한 횟수를 계산합니다. 이는 양식에 대한 최적의 표준 측정항목입니다. 왜냐하면 양식에 입력된 데이터를 수집하기 위해 양식이 처음 생성된 이유이기 때문입니다. 제출 버튼을 클릭하지 않으면 원하는 데이터를 얻을 수 없습니다.

제출 추세선에는 선택한 기간 동안 매일(또는 어제나 지난 24시간을 선택한 경우 시간당) 제출 버튼을 클릭한 횟수가 표시됩니다.

### 전망

조회수는 모든 자산(및 페이지)의 공통 측정항목입니다.

기간 필터와 함께 유용한 조회수는 특정 기간 동안 양식에 대한 조회수입니다. 고유한 IP 주소만 계산하려고 하지 않기 때문에 방문자 수와 동일하지 않습니다. 따라서 지난 30일 동안 한 명의 방문자(IP 주소)가 100번 페이지를 다시 방문할 수 있었습니다. 즉, 페이지 조회수는 100회이지만 방문자는 단 한 명입니다(매번 페이지에 액세스하는 데 동일한 장치를 사용했다고 가정). 그러나 사용자가 페이지와 다시 상호 작용하기 전에 페이지에서 최소 30분 동안 활동이 없으면 단일 사용자에 대한 고유 보기가 기록되지 않습니다.

### 포기

포기는 양식과 상호작용했지만 선택한 기간 동안 항목을 제출하지 못한 사용자의 일별(또는 어제 또는 지난 24시간을 선택한 경우 시간별) 비율입니다.

### 완료 시간

완료 시간은 양식 사용자가 양식과의 첫 상호 작용부터 제출 버튼을 누를 때까지 소요된 시간의 일일 평균(또는 어제 또는 지난 24시간을 선택한 경우 시간별)입니다.

방문자 행동 차트에 표시되는 시계열 지표는 시간 경과에 따른 자산 성과를 이해하는 데 가장 중요합니다. 하지만 자산 분석에는 더 많은 것이 있습니다.

## 청중

잠재고객 보고서는 차트를 사용하여 자산과 잠재고객의 상호작용에 대한 정보를 제공합니다. 다음 질문에 답합니다.

* 내 콘텐츠와 상호작용하는 사용자는 몇 명입니까?
* 이들 사용자 중 알려지거나 익명인 사용자는 몇 명입니까?
* 내 콘텐츠와 상호작용한 알려진 사용자 중 Analytics Cloud Segments에 속하는 사용자는 몇 명입니까?
* 세그먼트의 사용자 중 상위 15개 세그먼트는 무엇입니까?

잠재고객 보고서의 차트는 다음과 같습니다.

**제출:** 알려지거나 익명인 총 고유 방문자의 비율을 나타내는 도넛 차트입니다. 센터에는 총 순 방문자 수가 표시됩니다. 도구 설명에는 해당 섹션의 사용자 수(예: 알려진 사용자 또는 익명 사용자 수)가 표시됩니다.

!!! 참고
로그인 시 이메일 주소가 Analytics Cloud와 동기화된 사용자 데이터와 일치하면 개인이 알려진 것으로 간주됩니다. 신규 사용자가 사이트에 등록하면 해당 데이터가 Analytics Cloud와 동기화되고 알려진 개인으로 표시되는 데 최대 2시간이 걸릴 수 있습니다.

**분할된 제출:** 하나 이상의 Analytics Cloud 세그먼트에 속하거나 속하지 않는 알려진 개인의 비율을 나타내는 도넛 차트입니다. 중앙에는 알려진 개인의 총 수가 표시됩니다. 도구 설명에는 해당 섹션에 대해 알려진 개인의 수가 표시됩니다(예: 하나 이상의 세그먼트에 속하는 알려진 개인의 수).

**시청자 세그먼트:** 각 세그먼트를 구성하는 알려진 개인의 비율을 보여주는 막대 차트입니다. 차트에는 상위 5개 세그먼트 각각에 대한 막대가 표시되고 나머지 세그먼트는 마지막 막대에 집계됩니다. 마지막 막대의 도구 설명에는 나머지 각 세그먼트의 값이 표시됩니다.

![Forms audience metrics are visually presented with a variety of charts identifying unique visitors, known individuals, and known individual segments.](forms/images/03.png)

잠재고객에 대해 표시되는 세분화 데이터는 방문 당시 세그먼트에 속한 개인을 기반으로 합니다.

기본 장치 탭에서 운영 체제별(장치 유형별로 그룹화) 페이지 제출의 누적 막대 그래프를 봅니다.

![Forms users can also be viewed by technology.](forms/images/05.png)

선택한 기간 동안 상위 8개 웹 브라우저를 표시하는 도넛 차트를 보려면 **브라우저** 를 클릭하세요. 해당하는 경우 나머지 웹 브라우저는 9번째 도넛 세그먼트에 집계됩니다.

## 자산 표시 사이트


페이지 이름과 URL이 포함된 자산이 테이블에 나타납니다. Analytics Cloud의 모든 페이지 데이터와 마찬가지로 사이트 방문자가 어떤 방식으로든 상호작용한 페이지만 추적되고 보고됩니다. 링크를 클릭해도 자산이 있는 실제 페이지로 연결되지 않는다는 점에 유의하는 것이 중요합니다. 대신 페이지의 분석 보기로 연결됩니다. 여기에서 실제 페이지의 URL을 클릭할 수 있습니다.
