# ポータル製品とLiferayのカスタマイズ

**概要**

* Liferayは、簡単にカスタマイズできます。
* Liferayは、多くのオーディエンスとユースケースに優れたエクスペリエンスを提供します。

DXPを取り巻くベンダーの大部分は、コンテンツ管理システムのプロバイダーとしてスタートしたベンダーや、CRMやERPのバックグラウンドを持つベンダーで構成されています。 Liferayは、強力な水平ポータルの製品であるという点で異なります。

Liferayのポータル製品には、以下のような独自の視点があります。

* 強力な連携
* 顧客との関係性を重視
* 安全なデータの取り扱い
* デジタルの俊敏性

## 統合機能

デジタルエクスペリエンスプラットフォームにより、企業はデジタルトランスフォーメーションを活用できるようになります。 効果的にデジタルトランスフォーメーションを活用するには、異種システム間の障壁を取り払い、シームレスなエクスペリエンスを提供することです。

Liferayのようなポータル製品のプラットフォームは、バックエンドやレガシーシステムの統合に優れています。 これにより、顧客、従業員、その他のエンドユーザーは、企業内の複数のアプリケーションにわたって必要なデータにアクセスできるようになります。

例えば、CRMや企業のマーケティングオートメーションツールにロックされている顧客データは、ポータル製品のDXPを通じて簡単に統合・共有できます。 これにより、企業と顧客の時間と労力を節約するだけでなく、企業がターゲットとするオーディエンスをより深く理解し、サポートできます。

## 長期的な顧客との関係を重視

ポータルは、安全なインターフェースにより企業とターゲットオーディエンスを結びつけます。 パーソナライゼーションは、ユーザーが必要とするコンテンツや機能を提供します。 CMSテクノロジーは従来、匿名の見込み顧客にコンテンツを提供することに重点を置いてきたため、こうした認証されたエクスペリエンスには苦労する場合があります。

## 安全なデータ処理機能

CMSとは異なり、ポータルは多数のユーザーおよび情報の流れに対応し、多くの場合、顧客や企業の機密情報を処理します。 ポータル製品のDXPは暗号化されたシステムで構築されているため、データの保存と分析を同時に行い、安全でインサイトに満ちた情報管理を実現します。

## デジタルの俊敏性

ガートナー社が強調しているように、ポータルは共通のアーキテクチャフレームワークとして使用できます。 企業はWebサイトを構築し、作成したフレームワークをすぐに再利用して、より多くのタッチポイントを構築できます。 これにより、すべてのデジタルプレゼンスの機能と外観が統一され、ゼロからの再構築を回避することで時間とコストを節約できます。

ポータル製品のデジタルエクスペリエンスプラットフォームでソリューションを構築することで、顧客は個々のエクスペリエンスの品質を管理しながら、デジタル戦略を迅速に推進できます。

## カスタマイズの容易さ

Liferayは、最も柔軟で拡張性の高いDXPとして認められています。

Liferayのカスタマイズ方法は時代とともに変化し、柔軟性を維持しながら使いやすさを向上させてきました。 これまで、Liferayを拡張するプロセスは以下のように進化してきました。

* ソースの変更
* EXT環境
* プラグインSDK
* 拡張ポイント
* クライアント拡張

### 拡張機能の進化

当初、Liferayをカスタマイズするユーザーはソースコードから製品を構築し、あらゆるものを変更できました。 この優れた柔軟性は、独自のバグを持ち込む可能性があるためコストがかかり、新しいバージョンに簡単にアップグレードすることも困難でした。

進化するごとに、プラットフォームを拡張するために必要なナレッジだけでなく、同時に問題を引き起こす可能性を削減しました。 EXT環境は、Liferayのコードの上にユーザーコードをオーバーレイすることで、Liferayをカスタマイズしました。 これにより、ユーザーコードはLiferayのコードから分離されましたが、ソースを直接修正するのと同じ欠点が多くありました。 次にプラグインSDKが登場し、実行時にLiferayの動作を変更するインストール可能なファイルへのカスタマイズを保持するようになりました。

Liferay DXP 7.0では、OSGiを導入しました。 これにより、カスタマイズがモジュラーモデルに移行され、Liferayのコアコードから隔離された「コンテナ」に保管され、拡張ポイントを介して通信できるようになりました。 例えば、開発者はコマース・コンテキストに新しい支払い方法や注文ルール、またはドキュメントやメディアを保存するための新しいストレージメカニズムを実装できます。

しかし、こうした改良のすべてが、アップグレードを困難なものにしていました。 このプロセスにおける最新のステップは、最も強力で保守性の高い拡張メカニズムとして、クライアント拡張を導入することです。 クライアント拡張はLiferayのコアコードとは別に実行されるため、アップグレードが容易で、Liferay DXPをSaaSプラットフォームとして提供できるようにします。

## 多くのオーディエンスとユースケースにエクスペリエンスを提供

多くのDXPベンダーは、主にコーポレートサイトなどの消費者ベースのソリューションに重点を置いています。 Liferayのポータル製品により、複数のシステムからのデータと機能を組み合わせた認証されたエクスペリエンスを必要とするビジネスベースのソリューションに集中できます。

ビジネスに特化した認証ソリューションの例：

* カスタマーポータル
* サプライヤーポータル
* パートナーポータル
* 社内ポータルソリューション

上記はほんの一例です。

これらのB2Bソリューションはすべて、要件や統合のニーズに固有の複雑さを備えています。 この複雑さに対応できるソリューションに注力することで、Liferayは最も一般的なB2Cのユースケースにも対応できる体制を整えています。

　 これでLiferayが提案する独自の価値のモジュールは完了です。
